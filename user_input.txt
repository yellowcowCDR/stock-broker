•	모의투자 TR: 보통 **V**로 시작합니다. (예: 주식 현금 매수 VTTC0802U - Virtual의 V)
•	(참고: 시세 조회용 TR인 FHKST01010100 등은 공통으로 쓰는 경우가 많지만, 주문이나 계좌 조회 관련 TR은 반드시 T와 V를 구분해야 합니다.)
즉, 대련님의 스프링 부트 코드(RestTemplate 또는 WebClient 부분)에서 증권사로 요청을 보낼 때, 헤더의 tr_id 값에 실전투자용 코드가 하드코딩되어 있거나 분기 처리가 안 되어 있을 가능성이 높습니다.

3. 아키텍처 원칙

모든 구현은 Hexagonal Architecture 원칙을 준수해야 한다.

의존성 방향은 항상 외부 계층에서 내부 계층을 향해야 한다.

Adapter
  ↓
Application
  ↓
Domain

다음 구조를 기본으로 사용한다.

adapter
├── in
│   └── web
└── out
    ├── persistence
    └── external
application
├── port
│   ├── in
│   └── out
└── service
domain
├── model
└── exception

필수 규칙은 다음과 같다.

1. Controller는 JPA Repository 또는 JPA Entity를 직접 참조하지 않는다.
2. Controller는 반드시 Inbound Port인 UseCase만 호출한다.
3. Application Service는 Inbound Port를 구현한다.
4. Application Service가 DB 또는 외부 API를 사용할 때는 Outbound Port를 호출한다.
5. Persistence Adapter가 Outbound Port를 구현한다.
6. Domain 계층은 Spring, JPA, HTTP에 의존하지 않는다.
7. Application 계층은 HttpStatus, ResponseEntity에 의존하지 않는다.
8. JPA Entity는 adapter.out.persistence 내부에서만 사용한다.
9. Domain Model과 Web DTO는 가능한 불변 객체 또는 Java Record로 작성

작업지시서 수정 사항 (Task 1 변경)

기존 작업지시서의 Task 1 (관심 종목 탐색 API) 의 목적과 응답 모델을 아래 내용으로 수정하여 구현합니다.

이번 수정의 핵심은 Watchlist를 단순한 인기 종목 또는 급등 종목 목록이 아닌, AI가 후속 분석을 수행하기 위한 시장 분석 후보군(Market Analysis Candidates)으로 정의하는 것입니다.

⸻

변경 목적

기존 Watchlist는 단순한 관심 종목 목록을 반환하는 형태였습니다.

하지만 실제 자율형 퀀트 에이전트에서는 이미 상승한 “핫한 종목”을 따라가는 방식만으로는 지속적인 성과를 기대하기 어렵습니다.

따라서 Watchlist는 매수 대상이 아니라 후속 분석 대상을 선정하는 1차 필터로 사용하도록 변경합니다.

즉,

전체 시장
    ↓
Watchlist 생성
    ↓
기술적 분석
    ↓
뉴스 분석
    ↓
리스크 분석
    ↓
매매 전략 수립
    ↓
주문

의 첫 단계 역할을 수행해야 합니다.

⸻

Task 1 목적 변경

기존

에이전트가 당일 분석할 관심 종목 후보군을 반환합니다.

다음으로 변경합니다.

에이전트가 당일 분석할 시장 분석 후보군(Market Analysis Candidates)을 반환합니다.

이 API는 단순 인기 종목이나 급등 종목을 추적하는 기능이 아니며, 다양한 투자 전략 관점에서 후속 분석이 필요한 종목을 선별하기 위한 API입니다.

⸻

Watchlist의 역할

Watchlist는 매수 신호가 아닙니다.

Watchlist는 이후 단계에서

* 현재가 조회
* 기술적 분석
* 뉴스 감성 분석
* 포트폴리오 리스크 분석

등을 수행하기 위한 분석 대상 목록입니다.

Application Service 및 AI Agent는 Watchlist 결과만으로 주문을 수행해서는 안 됩니다.

작업지시서 변경 사항 (실제 시장 데이터 기반으로 변경)

기존 작업지시서에서 Mock 데이터 기반 구현을 아래 내용으로 변경합니다.

이번 변경의 목적은 Hermes가 한국투자증권(KIS) 모의투자 환경을 이용하여 실제 시장 데이터로 학습하도록 하는 것입니다.

모의투자는 주문과 계좌만 가상이며, 시장 데이터는 실제 데이터를 사용해야 합니다.

⸻

변경 목적

기존 작업지시서에서는 일부 기술적 지표와 뉴스를 Mock 데이터로 생성하도록 작성되어 있었습니다.

이를 모두 제거합니다.

Hermes는 실제 시장 환경과 동일한 데이터를 기반으로 판단하고 학습해야 합니다.

즉,

실제 시장 데이터 조회
        ↓
기술적 분석
        ↓
뉴스 분석
        ↓
전략 적용
        ↓
모의투자 주문
        ↓
체결 결과 분석
        ↓
전략 개선

의 흐름으로 동작해야 합니다.
